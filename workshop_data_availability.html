tidycensus: Your Data Access Tool

You set up your Census API key in the previous workshop. Now we use it.

tidycensus gives you ACS data directly in R — no manual downloading.

library(tidycensus)
library(tidyverse)

get_acs(
  geography = "county",      # unit of analysis
  variables = "B19013_001",  # which variable
  state     = "VA",          # which state
  year      = 2022,          # which year
  survey    = "acs5"         # 5-year estimates
)

Finding Variable Codes

ACS variables follow a pattern: Table + Cell number

B19013_001 = Table B19013 (Median Household Income), Cell 001


# Search the full variable list
vars <- load_variables(2022, "acs5", cache = TRUE)
View(vars)

# Search by keyword
vars |>
  filter(str_detect(label, "income|Income")) |>
  select(name, label, concept)

💡 Always check the concept column — it tells you which table the variable belongs to.

Pulling Multiple Years

Use map_dfr() to loop over years and stack the results into one dataframe:

years <- 2015:2022

multi_year <- map_dfr(years, function(yr) {
  get_acs(
    geography = "county",
    variables = c(
      median_income = "B19013_001",
      poverty       = "B17001_002"
    ),
    state  = "VA",
    year   = yr,
    survey = "acs5"
  ) |>
    mutate(year = yr)
})

map_dfr() = run this for each year, then stack all results into one dataframe.

The Problem: Not All Variables Exist

Try pulling broadband access for 2012:

get_acs(
  geography = "county",
  variables = "B28002_004",  # broadband access
  state     = "VA",
  year      = 2012,
  survey    = "acs5"
)

You get an error. ACS didn’t track broadband at the county level before 2017.

This is not a data error — it’s measurement history. And you need to know this before you build your analysis.

The Audit Function

Use tryCatch() to handle errors gracefully — returns TRUE if data exists, FALSE if not:

check_variable <- function(var, year) {
  tryCatch({
    result <- get_acs(
      geography = "county",
      variables = var,
      state     = "VA",
      year      = year,
      survey    = "acs5"
    )
    nrow(result) > 0
  }, error = function(e) FALSE)
}

check_variable("B28002_004", 2012)  # FALSE
check_variable("B28002_004", 2020)  # TRUE

Running the Full Audit

vars <- c(
  "B19013_001",  "B17001_002",  "B15003_022",
  "B27001_001",  "B01003_001",  "B28002_004"
)

years <- 2010:2023

audit_results <- expand_grid(variable = vars, year = years) |>
  mutate(available = map2_lgl(variable, year, check_variable))

⚠️ This makes ~84 API calls — takes a few minutes. Save immediately:

write_csv(audit_results, "audit_results.csv")

Adding Labels and Categories

Variable codes aren’t human-readable. Add your own labels and categories:

var_metadata <- tibble(
  variable = c("B19013_001","B17001_002","B15003_022",
               "B27001_001","B01003_001","B28002_004"),
  label    = c("Median Household Income","Population Below Poverty",
               "Bachelor's Degree","Health Insurance Universe",
               "Total Population","Broadband Access"),
  category = c("Socioeconomic","Socioeconomic","Socioeconomic",
               "Healthcare Access","Demographics","Infrastructure")
)

audit_final <- audit_results |>
  left_join(var_metadata, by = "variable") |>
  group_by(variable, label, category) |>
  summarise(
    n_years       = sum(available),
    years_present = list(year[available == TRUE]),
    missing_years = list(year[available == FALSE]),
    .groups = "drop"
  )

Building the Dashboard

Option 1 — Pure R using reactable:

library(reactable)
library(htmlwidgets)

widget <- reactable(audit_final, filterable = TRUE, searchable = TRUE)
saveWidget(widget, "data_availability.html")

Option 2 — R + Claude for a more polished result:

library(jsonlite)
cat(toJSON(audit_final |>
  mutate(years_present = map(years_present, as.integer),
         missing_years = map(missing_years, as.integer)),
  pretty = TRUE))
